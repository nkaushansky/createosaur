# Acceptance Checklists

## A. Master illustration checklist

- [ ] Species identity is clear
- [ ] Creature belongs visually with the approved T. rex
- [ ] Full-body profile is clean and readable
- [ ] Head looks credible and animal-like
- [ ] Eye is restrained
- [ ] Mouth is closed or nearly closed
- [ ] Pose is stable and riggable
- [ ] No scenery clutter
- [ ] No fake labels
- [ ] No childish/cartoon drift

## B. Layer extraction checklist

- [ ] Visible result matches approved master
- [ ] Canonical anatomical layers are present
- [ ] Pieces look useful for a rig
- [ ] Full-canvas alignment is preserved
- [ ] Hidden overlap exists beneath adjoining parts
- [ ] No accidental redesign happened during extraction
- [ ] No fake diagram text
- [ ] No clutter unrelated to the pipeline

## C. Hidden overlap checklist

- [ ] Head overlaps neck correctly
- [ ] Neck overlaps torso correctly
- [ ] Torso overlaps pelvis correctly
- [ ] Pelvis overlaps tail correctly
- [ ] Thighs overlap body correctly
- [ ] Lower legs overlap thighs correctly
- [ ] Overlap seems sufficient for modest motion
- [ ] Overlap does not visibly distort the neutral assembled creature

## D. Pattern mask checklist

- [ ] Solid mask exists per layer
- [ ] Mottle mask exists per layer
- [ ] Bands mask exists per layer
- [ ] Masks are grayscale
- [ ] Masks are clipped to the exact layer
- [ ] Masks align in neutral pose
- [ ] Masks appear suitable for moving with the part
- [ ] No screen-space-only body-wide pattern logic is implied

## E. Trait pack checklist

- [ ] Trait style matches approved dinosaur family
- [ ] Traits fit the base animal believably
- [ ] Front/back split pieces exist where needed
- [ ] Attachment shadows exist where needed
- [ ] Important base features remain visible
- [ ] Full redesign did not sneak in
- [ ] Optional assembled preview remains coherent

## F. Claude Code integration checklist

- [ ] The approved asset package is complete enough to implement
- [ ] Exact-source or approved-source files are clearly named
- [ ] Manifest/index exists or can be created cleanly
- [ ] Repo integration remains bounded
- [ ] Production renderer is not silently modified
- [ ] Deterministic pose evaluation remains testable
- [ ] Review screenshots can be compared back to the approved master
